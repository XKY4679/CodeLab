===========================================================
       批量视频压缩工具 2.0 —— 使用教程（面向小白）
===========================================================

【一、这个工具能干嘛？】

把视频文件批量压缩变小，方便传输、上传或节省硬盘空间。
支持两种压缩模式：
  - 质量预设：选"高/中/低"质量，程序自动决定压缩程度
  - 指定目标大小：比如"每个文件压到 100MB 左右"

压缩完成后会显示每个文件的 原始大小 → 压缩后大小 → 节省百分比。
原文件不会被删除，压缩后的文件保存在单独的"压缩输出"文件夹里。


【二、前提条件：安装 ffmpeg】

本工具依赖 ffmpeg（免费的视频处理引擎），必须先安装。

安装步骤：
  1. 打开 https://ffmpeg.org/download.html
  2. 下载 Windows 版本（选 "Windows builds from gyan.dev"）
  3. 解压后，把里面 bin 文件夹的路径加到系统 PATH
     - 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
     - 在 Path 里新增一行，填 ffmpeg 的 bin 目录路径
  4. 打开 cmd，输入 ffmpeg -version，有输出就说明安装成功

Python 无需额外安装库，程序只用了自带的 tkinter。

（如果嫌麻烦，也可以搜索"ffmpeg 安装教程"，网上有很多图文教程）


【三、运行程序】

方法 1：双击 VideoCompression_2.0.py
方法 2：命令行输入  python VideoCompression_2.0.py

启动后程序会自动检测：
  - 是否安装了 ffmpeg（没装会提示）
  - 是否有 NVIDIA 显卡可以加速（有就自动用，没有就用 CPU，都能用）


【四、操作步骤】

1. 点击「选择视频文件夹」
   - 选择包含视频的文件夹
   - 程序会扫描并列出所有视频文件和各自的大小

2. 选择压缩模式：

   ▶ 模式一：质量预设
     - 高质量（体积较大）：画质几乎无损，文件缩小有限
     - 均衡（推荐）：画质和体积的最佳平衡，日常使用首选
     - 小体积（画质有损）：文件会很小，但画质有明显下降

   ▶ 模式二：指定目标大小
     - 填入一个数字，比如 100，表示每个视频压缩到约 100MB
     - 如果原文件已经小于目标值，会自动跳过（不做无用压缩）
     - 注意：实际大小可能和目标有±10%左右的偏差，这是正常的

3. 点击「开始压缩」
   - 程序会逐个处理，实时显示进度
   - 每个文件处理完会显示：原始大小、压缩后大小、节省比例
   - 单个文件失败不影响其他文件继续

4. 压缩完成后
   - 底部显示总计大小对比和节省比例
   - 点「打开输出文件夹」直接查看压缩好的文件
   - 压缩后的文件在原文件夹下的"压缩输出"子文件夹里


【五、支持的视频格式】

MP4、MOV、AVI、MKV、M4V、WMV、FLV、WebM


【六、关于 GPU 加速】

  - 如果电脑有 NVIDIA 显卡（GTX/RTX 系列），程序会自动使用
    GPU 硬件加速（h264_nvenc），压缩速度快很多
  - 如果没有 NVIDIA 显卡，会自动回退到 CPU 编码（libx264），
    速度慢一些但压缩效果一样好
  - 不需要手动设置，程序全自动检测


【七、常见问题】

Q：提示"未检测到 ffmpeg"？
A：请确认 ffmpeg 已安装并加入系统 PATH（见上面安装步骤）。

Q：压缩速度很慢？
A：CPU 编码本身就比较慢，尤其是大视频文件。
   如果有 NVIDIA 显卡，确保安装了最新驱动，程序会自动加速。

Q：指定 100MB 但实际压出来 110MB？
A：正常现象。ffmpeg 按目标码率压缩，实际大小会有小幅偏差。
   视频内容越复杂（如动作片），偏差可能越大。

Q：原视频会被删除吗？
A：不会！压缩后的文件保存在"压缩输出"文件夹里，原文件不动。

Q：压缩后画质很差？
A：如果用的是"小体积"预设或目标 MB 设得太低，画质会下降。
   建议先用"均衡"预设试试，或者适当调大目标 MB。
